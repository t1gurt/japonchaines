# サイト構造分析

## 概要
このプロジェクトは、**App Router** (`src/app`) を使用した Next.js アプリケーションです。スタイリングには TypeScript と Tailwind CSS が使用されています。

## ディレクトリ構造
- `src/app`: メインのアプリケーションルートとレイアウトが含まれています。
- `src/components`: 再利用可能な UI コンポーネントが含まれています。
- `src/lib`: ユーティリティ関数と共有ロジックが含まれています。
- `src/styles`: グローバルスタイルが含まれています。
- `public`: 静的アセット（画像、アイコン）が含まれています。

## サイトマップとルーティング
アプリケーションは、いくつかの主要なカテゴリーを中心に構成されています：

### 1. メインページ
- `/`: ホームページ
- `/a-propos`: 概要ページ
- `/contact`: お問い合わせページ

### 2. コンテンツカテゴリー
- **`/chaines`**: レストランチェーンの一覧。
    - `/chaines/[slug]`: 個別のチェーン詳細（例：`/chaines/anan`）。
    - **ルーティング**: 静的ルート。各チェーンには独自のディレクトリと `page.tsx` があります。
- **`/type-plat`**: 料理カテゴリー（料理の種類）の一覧。
    - `/type-plat/[slug]`: 個別の料理カテゴリー詳細（例：`/type-plat/ramen`）。
    - **ルーティング**: 静的ルート。各カテゴリーには独自のディレクトリと `page.tsx` があります。
- **`/guides`**: 実用的なガイドの一覧。
    - `/guides/[slug]`: 個別のガイド詳細。
    - **ルーティング**: 静的ルート。
- **`/tutorial`**: インタラクティブなチュートリアルセクション。
    - **実装**: `TutorialDashboard` コンポーネントを使用しています。

## ルーティング戦略
- **静的ルーティング**: プロジェクトでは主に静的ルーティングが使用されており、各ページはファイルシステム内で明示的に定義されています（例：`src/app/chaines/anan/page.tsx`）。
- **動的ルーティング**: 現在の構造では、動的ルートセグメント（`[slug]`）の使用は限定的です。ほとんどの「動的」コンテンツは、実際には個別の静的ファイルにハードコードされています。

## 主な観察事項
- **ハードコードされたデータ**: コンテンツ（メニュー項目、説明、リスト）の多くは、データベースや CMS から取得されるのではなく、`page.tsx` ファイル内に直接ハードコードされています。
- **繰り返されるレイアウト**: 類似したページ（例：チェーン詳細）は共通のレイアウト構造を共有していますが、個別に実装されているため、コードの重複が発生しています。
